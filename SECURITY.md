# 安全策略

## API 密钥安全

本项目是纯前端应用。API 密钥存储在浏览器的 `localStorage` 中，**不会**上传到任何服务器或包含在源代码中。

**注意事项：**
- 切勿将你的 API 密钥提交到 Git 仓库。
- 在公共或共享设备上使用后，建议在浏览器设置中清除站点数据。
- 如果你部署了公开版本并预置了 API 密钥，请注意前端代码中的密钥对用户可见。

## 报告漏洞

如果你发现了安全漏洞，请**不要**在公开 Issue 中报告。

请通过以下方式私密报告：
1. 使用 GitHub 的 [Private vulnerability reporting](../../security/advisories/new) 功能。
2. 或发送邮件至项目维护者（请在仓库描述中查找联系方式）。

我们会在 **72 小时**内确认收到你的报告，并在 **7 个工作日**内提供修复计划。
