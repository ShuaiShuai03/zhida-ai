  ## 摘要

  本次范围按最严格口径执行：覆盖根文档、.github 模板、index.html 内联帮助文案，以及代码中的行为性注释/JSDoc；同时把审查扩展为整个仓库缺陷审计。已
  确认的问题如下：

  - README.md:6-8,20,74 与 SECURITY.md:5,16-20 的安全/兼容性表述失真；实际代码会把 API key 直接发往用户配置的 API 与 /v1/models（js/api.js:53-
    60,155-156），且当前自定义 sanitizer 只是基础清洗，不足以支撑“防 XSS”的绝对承诺（js/utils.js:63-145）。
  - 流式请求的超时与中止语义有缺陷：超时路径先触发 onError 再因 AbortError 触发 onDone，会导致空 AI 消息、重复状态收尾或错误/成功并存（js/
    api.js:25-28,126-139，js/chat.js:218-275,335-390）。
  - “获取模型列表”在未保存设置时直接污染运行态 API 配置，而且代码注释承诺“失败后恢复”但未实现（js/app.js:537-557）。
  - 会话列表只在加载时排序，后续保存不重排，导致最近活跃对话不会上浮（js/storage.js:72-78,98-105）。
  - 导出 Markdown 时使用当前全局选中模型而非会话自身模型，导出元数据会写错（js/chat.js:459-469）。
  - 首条消息若仅包含附件，标题会被写成空字符串（js/chat.js:153-156）。
  - 文件上传支持集与 <input accept> 不一致，bat/env/jsx/kt/ps1/r/scala/swift/tsx 等“代码里支持、文件选择器里不支持”（js/app.js:61-67，
    index.html:151-155）。
  - 文档链接与克隆命令存在仓库身份错误或脆弱写法：README 仍使用 your-username，根文档使用 ../../issues / ../../security/... / ../../actions/... 这
    类 GitHub UI 相对路径（README.md:3,27,52，CONTRIBUTING.md:9，SECURITY.md:17）。
  - 安全与社区流程文档缺少仓库内可执行的报告渠道：SECURITY.md 要求邮件联系但仓库内没有联系方式，且“历史 Release 支持”与实际无 tag/release 不符；
    CODE_OF_CONDUCT.md 也没有明确举报入口（SECURITY.md:16-27，CODE_OF_CONDUCT.md:21-27）。
  - README 未覆盖已实现的关键能力与运行前提：文件上传、拖拽/粘贴图片、清空所有对话、手动拉取模型、浏览器直连 API 需要 CORS、运行时依赖 CDN 资源
    （index.html:75-85,151-155,202-203,301-304，js/app.js:204-229,537-557，js/api.js:55-60,155-156）。
  - 应用内快捷键帮助文案对 macOS 是错的：UI 只显示 Ctrl，实现实际使用 Cmd/Meta（index.html:281-290，js/app.js:597-618）。
  - 存在明确的可访问性缺陷：禁用页面缩放、对话框缺少 aria-modal、表单帮助文字未与字段关联、模型选择器 aria-expanded 不随展开状态同步、列表/列表框
    语义不完整（index.html:5,69,104-109,174,243,262,186-231）。
  - 贡献与协作文档互相打架：CONTRIBUTING.md 要求 Safari 测试，但 PR 模板未要求；Bug 模板缺少 API 提供商/Base URL/模型/CORS/Console 等定位必需信息
    （CONTRIBUTING.md:22，.github/PULL_REQUEST_TEMPLATE.md:20-24，.github/ISSUE_TEMPLATE/bug_report.md:27-33）。
  - 内联文档本身也有漂移：showToast JSDoc 仍写默认 3000ms，但实现是 1000ms（js/ui.js:19-24）；“失败后恢复旧配置”的注释与实现不一致（js/app.js:541-
    556）。
  - CI 只做非常浅的结构检查；JSON 校验甚至只输出 WARN 不失败，无法兜住文档链接、浏览器行为、无障碍和上述运行时回归（.github/workflows/ci.yml:52-
    56）。

  ## 关键修改

  - 先修运行时缺陷：重构 streamChatCompletion 的终态协议，拆分 onDone / onAbort / onError 或返回显式终止原因；规则固定为“手动停止仅在已有非空输出
    时保留部分回答，超时/失败绝不写入 AI 成功消息”。同时修复会话重排、导出模型来源、附件首消息标题回退。
  - 收敛设置与模型拉取流程：让 fetchAvailableModels 接受显式凭据参数，不再偷改全局 state；在设置弹窗内使用“暂存配置 + 暂存模型列表”，只有点击“保存
    设置”后才提交 API 地址、密钥与模型缓存。
  - 消除配置单源失真：把支持的文本扩展名和 accept 字符串统一为一个来源，在初始化时由 JS 注入到文件输入框；把默认系统提示词、toast 默认时长等重复常
    量集中复用，移除硬编码字符串比较。
  - 修正文档与内联帮助：README、SECURITY、CONTRIBUTING、CODE_OF_CONDUCT、Issue/PR 模板、快捷键弹窗统一改为基于当前仓库事实的文案。GitHub UI 链接改
    为仓库全路径；克隆地址改为 ShuaiShuai03/zhida-ai；删除无凭据支撑的邮箱/SLA/历史 release 承诺，默认仅保留仓库内真实存在的报告渠道。
  - 补全文档缺口：README 新增“浏览器直连 API 的 CORS 前提”“运行依赖外部 CDN 或需要本地 vendoring”“文件上传/拖拽/粘贴限制”“清空对话”“手动获取模型列
    表”“GitHub Pages/静态部署的适用边界”。
  - 处理安全表述与实现脱节：默认方案不是只改文案，而是同时把 sanitizeHTML 替换为经过审计的现成 sanitizer（优先本地 vendored/pinned 版本，避免再增
    选择器改为原生按钮语义并同步 aria-expanded/aria-controls；对列表结构改用原生 ul/li/button 或删除错误 ARIA 角色；快捷键帮助改为运行时按平台渲
  - 升级质量门禁：在 CI 中新增文档链接检查、JS 语法检查、仓库事实断言（克隆地址/安全文案/模板字段）、以及一组浏览器级 smoke tests，覆盖“停止/超
    时”“模型拉取不污染状态”“会话排序”“附件上传入口”“导出元数据”“快捷键文案/行为”。
  ## 接口与行为变更

  - js/api.js 内部接口需要变更：streamChatCompletion 不再把 AbortError 混同为成功结束；fetchAvailableModels 改为可接收显式配置对象，而不是只读全局
    state。
  - 用户可见行为会变化：停止生成与超时不再产生空 AI 消息；附件首条消息会生成可读标题；导出文件的模型信息与原会话一致；文件选择器会真正暴露所有受支
    持文本类型；快捷键帮助会按平台显示 Ctrl/Cmd。
  - 安全文案会更精确：明确“密钥仅保存在浏览器本地，但会被发送到你配置的 API 服务”；明确“仅支持可被浏览器直接访问且允许 CORS 的 OpenAI 兼容接口”；
    明确 CDN/本地 vendoring 依赖。

  ## 测试计划

  - 为流式会话新增用例：成功、手动停止、超时、网络失败四类路径分别验证消息落库、UI 收尾和按钮状态，确保不会再出现空 AI 消息或双重终态。
  - 为设置流程新增用例：在设置弹窗中点击“获取模型列表”后取消/失败/成功未保存三种场景，验证全局 API 配置与当前会话状态不被污染。
  - 为数据一致性新增用例：发送消息、重命名、重新生成后侧栏排序必须按 updatedAt 即时变化；导出必须写会话自身模型；附件首条消息标题必须回退到默认标
    题或附件名摘要。
  - 为上传入口新增用例：accept 列表与支持扩展常量一致；文件选择、拖拽、粘贴三种入口都能覆盖图片与文本文件。
  - 为文档新增检查：README/SECURITY/CONTRIBUTING/.github 模板中的仓库链接、克隆地址、报告渠道、平台快捷键、功能列表与实现事实一致。
  - 为无障碍新增检查：页面可缩放、对话框 aria-modal/焦点循环/焦点恢复、表单描述关联、展开控件状态同步、键盘可达性和平台化快捷键提示。
  - 为 CI 新增回归门禁：文档链接失败即失败、JSON 无效即失败、静态资源/脚本语法失败即失败、浏览器 smoke 失败即失败。
  - 对仓库内不存在的维护者邮箱/私密联系方式，不做臆造；默认删除相关承诺，仅保留仓库内真实存在且可执行的报告通道，直到维护者提供明确信息。
  - 对当前自定义 HTML 清洗逻辑，默认采用“替换为成熟方案 + 同步收紧文档”的双修复，而不是只做措辞降级。
  - 对“停止生成”的产品语义，默认保留非空部分输出；对“超时/错误”语义，默认不生成成功消息。
